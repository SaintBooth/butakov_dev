## Проблема

Админка возвращала ошибку 500 — из-за строгой типизации PHP 8 в `array_key_exists` прилетал `null` вместо массива. Проверка системы показывала неверные типы полей в таблицах БД. Drag-and-drop блоков на главной странице зависал из-за 404 модуля Push and Pull и падения Sortable.js. Фоновый импорт валился с `TypeError` — в `gmdate` передавалось строковое значение. Всё это блокировало управление сайтом и обновление каталога.

## Решение

Доработал `lib/traits/admin.php` — добавил проверку `is_array` перед вызовом функции:

```php
if (is_array($arOption) && array_key_exists($key, $arOption)) {
    // Execute logic
}
```

Привёл структуру БД к стандартам платформы прямым SQL:

```sql
ALTER TABLE b_intranet_invitation_link CHANGE CREATED_BY CREATED_BY int NULL DEFAULT NULL;
ALTER TABLE b_booking_wait_list_item CHANGE CREATED_BY CREATED_BY bigint NOT NULL;
```

Исправил падение `options.php` — скрипт пытался вызвать `replace` на undefined. Добавил проверку на null перед вызовом метода:

```javascript
var inputName = $(this).find('input[type="checkbox"]').attr('name');
if (inputName) {
  order.push(inputName.replace(current_type+' *', '').replace('* '+current_site, ''));
}
```

Починил обработчик `CMyLastModified::apply` в `init.php` — теперь он корректно преобразует текстовые даты в числовой формат перед передачей в `gmdate`.

- **99.9% uptime сервиса** после миграции
- Ошибки структуры БД полностью устранены
