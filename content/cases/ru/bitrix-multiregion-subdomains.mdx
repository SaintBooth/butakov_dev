## Проблема

Проект на 1С-Битрикс с шаблоном Аспро: Максимум. Хостинг Timeweb Cloud, классическая связка Nginx + Apache. Задача на словах простая: региональные версии сайта для 27 городов России на поддоменах, с нормальным SEO и без поломанного пользовательского пути.

В Аспро региональность вроде бы есть из коробки. Я включил её, развернул поддомены и пошёл проверять. Дальше начался список проблем.

Запросы к новым поддоменам вообще не доходили до сайта — их перехватывал дефолтный блок Nginx и отдавал заглушку BitrixVM. Попап выбора города подставлял ссылки на основной домен вместо поддоменов. Я долго не мог понять почему. Оказалось, компонент Аспро ждёт определённого заполнения свойств инфоблока, и об этом нигде внятно не написано.

Самое неприятное нашлось потом. Блог и новости компании открывались по всем 27 адресам. Посчитайте сами: каждый раздел умножается на 27 городов. Сотни тысяч дублей, и поисковые роботы тратят краулинговый бюджет на копии одной и той же статьи. За такое Яндекс по голове не погладит.

Ещё пользовательский путь. Человек сидит в каталоге своего города, уходит почитать статью в блог на основном домене, возвращается в каталог. И попадает в каталог основного домена, без своего города. Система про него просто забыла. И мелочь напоследок: ссылки на общие разделы в карточках товаров вели на поддомены и собирали лишние 301-е редиректы. Ссылочный вес утекал на ровном месте.

## Решение

Начал с инфраструктуры. Настроил Wildcard DNS (`*.domain.ru`) и написал конфигурацию Nginx, которая проксирует запросы поддоменов на Apache с передачей оригинального `$host`. Без этого заголовка ядро Битрикса не понимает, на каком поддомене находится. Одна строчка, а без неё ничего не работает. Дубли закрыл там же, в Nginx: любой запрос к `/blog/` или `/company/news/` на поддомене получает 301 на главный домен. PHP и база данных в этом даже не участвуют, редирект отрабатывает мгновенно.

```nginx
server {
    server_name ~^(?<city>[\w-]+)\.domain\.ru$;

    location ~ ^/(blog|company/news)/ {
        return 301 https://domain.ru$request_uri;
    }

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
    }
}
```

Для возврата пользователя в свой город написал обработчик события `OnPageStart`. Логика такая: если человек оказался на основном домене в локальном разделе вроде `/catalog/`, скрипт читает ID региона из `$_COOKIE['current_region']`, находит поддомен в инфоблоке регионов и отправляет пользователя туда. GET-параметры при этом сохраняются, так что выбранные фильтры и страница пагинации не слетают. Отдельно запретил скрипту срабатывать в AJAX-запросах, REST API и CLI. Проверил на горьком опыте: без этой защиты обработчик ломает импорт.

```php
public static function returnToRegion(): void
{
    if (self::isAjaxOrApi() || php_sapi_name() === 'cli') {
        return;
    }

    $regionId = (int)($_COOKIE['current_region'] ?? 0);
    $subdomain = self::getRegionSubdomain($regionId);

    if ($subdomain && self::isLocalSection()) {
        LocalRedirect('https://' . $subdomain . $_SERVER['REQUEST_URI'], true, '302 Found');
    }
}
```

Оставались внутренние ссылки на блог и новости в вёрстке. Тут помог обработчик `OnEndBufferContent`. Он регулярными выражениями проходит по итоговому HTML перед отдачей браузеру и заменяет региональные ссылки абсолютными ссылками главного домена. Роботы Яндекса и Google видят чистые прямые ссылки, внутри сайта ни одного лишнего редиректа.

```php
public static function fixGlobalLinks(string &$content): void
{
    $content = preg_replace(
        '#https?://[\w-]+\.domain\.ru(/(?:blog|company/news)/)#',
        'https://domain.ru$1',
        $content
    );
}
```

Весь код вынес из `init.php` в отдельный статический класс `regional_handlers.php`. Файл `init.php` в проектах на Битриксе и так вечно превращается в свалку. Изолированный класс проще читать, он переживает обновления ядра и шаблона, и его легко расширять.

## Результат

- **0 контентных дублей** — роботы видят чистые прямые ссылки
- Часть нагрузки переехала с PHP на быстрый Nginx
- Пользователь свободно ходит между глобальным блогом и каталогом своего города
- Новый город, хоть 28-й, хоть 50-й, добавляется **одним элементом инфоблока**
