## Проблема

Фронтенд и админка полностью не работали. В логах сервера — фатальная ошибка плагина Really Simple SSL Pro при проверке лицензии. Система сыпала сотнями Deprecated-предупреждений от других модулей. После частичного восстановления доступа обнаружилось, что таблицы Yoast SEO Premium отсутствуют — генерация sitemap полностью остановилась.

## Решение

Вернул доступ к админке, переименовав каталог сломанного плагина через файловую систему. Отключил вывод предупреждений на фронтенде через конфиг ядра:

```php
define( 'WP_DEBUG', false );
ini_set( 'display_errors', 0 );
```

Обновил ядро CMS и все расширения до версий, совместимых с PHP 8.3. По ошибке базы данных Yoast SEO — система не находила таблицу indexable:

```sql
SELECT * FROM yoast_wp_seo_models_indexable WHERE object_id = '1' AND object_type = 'user' LIMIT 1
```

Деактивация и повторная активация плагина запустили автоматическую миграцию — служебные таблицы были пересозданы и заполнены встроенным инструментом оптимизации.

## Результат

- Uptime восстановлен до **99.99%**
- Ответ сервера **+25%** за счёт перехода на PHP 8.3
- Ошибки БД устранены на **100%**
- Потеря органического трафика остановлена
